## Recording Checklist

### Pre-Production

- [ ] Write full script (word-for-word)
- [ ] Practice 5+ times (aim for smooth delivery)
- [ ] Time yourself (must be under 3:00)
- [ ] Prepare demo environment:
  - [ ] Clean browser (no tabs, bookmarks)
  - [ ] Clear terminal history
  - [ ] Test all features work
  - [ ] Have backup recordings if live demo fails

### Production

**Equipment:**
- Screen recorder: QuickTime (Mac), OBS (Windows/Linux)
- Microphone: AirPods or better (no laptop mic!)
- Resolution: 1920√ó1080 minimum
- Frame rate: 30fps or 60fps

**Settings:**
- Turn off notifications (Slack, email, etc.)
- Close unnecessary apps (show only demo)
- Hide desktop icons (clean background)
- Use incognito/private browsing (no autofill surprises)

**Recording Tips:**
- Record in 4K if possible (downscale to 1080p for upload)
- Record audio separately (easier to fix mistakes)
- Leave 5 seconds of silence at start/end (for editing)
- Record each feature separately (easier to re-record if needed)

### Post-Production

**Editing Tools:**
- Free: iMovie (Mac), DaVinci Resolve (all platforms)
- Paid: Final Cut Pro, Adobe Premiere

**Edits to Make:**
- Trim silence at start/end
- Add background music (low volume, no lyrics)
- Add text overlays for key points
- Speed up slow parts (loading screens) to 1.5-2x
- Add timestamps/chapter markers
- Color correct (if needed)

**Export Settings:**
- Format: MP4 (H.264)
- Resolution: 1920√ó1080
- Bitrate: 8-10 Mbps
- Audio: AAC, 192 kbps

## Common Mistakes to Avoid

### ‚ùå Don't Do This:

1. **Long intro** - "Hi, my name is... I built this because..."
   - ‚úÖ **Do instead:** Hook in first 10 seconds

2. **Live coding** - "Let me show you the code..."
   - ‚úÖ **Do instead:** Show code for 3 seconds max, keep moving

3. **Apologizing** - "Sorry, this might not work..."
   - ‚úÖ **Do instead:** Confidence! Pre-record if nervous

4. **Reading slides** - "As you can see on this slide..."
   - ‚úÖ **Do instead:** Show, don't tell (visuals speak)

5. **Technical jargon** - "We use a RESTful CRUD API with..."
   - ‚úÖ **Do instead:** "Users can create, read, update meals via API"

6. **Slow demos** - Wait 10 seconds for page to load...
   - ‚úÖ **Do instead:** Speed up video during loading (1.5-2x)

7. **No audio** - Silent screencasts with captions
   - ‚úÖ **Do instead:** Voiceover explaining what's happening

8. **Over 3 minutes** - "Just one more feature..."
   - ‚úÖ **Do instead:** Ruthlessly cut to 2:45 (safety margin)
